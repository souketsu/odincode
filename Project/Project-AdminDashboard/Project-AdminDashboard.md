# 管理后台仪表盘开发项目

奥丁计划（The Odin Project）助力每一位心怀热忱的准网页开发者，免费开启协同学习之旅。

## 一、项目介绍

你已经积累了大量 CSS Grid 布局的实战经验，接下来我们要着手开发一个完整的管理后台仪表盘。在这个项目中，你可以灵活使用各类开发工具，但建议优先采用 Grid 完成大部分布局工作。如果需要回顾相关知识，可以重温之前的课程或练习任务。

## 二、任务要求

### （一）第一步：项目准备与规划

搭建 Git 代码仓库（如果忘记操作流程，可以参考之前的项目）。

创建 HTML 和 CSS 文件，并写入一些测试内容，验证文件之间是否正确关联。

下载 项目设计稿的高清版本，梳理整体布局结构，规划 HTML 文档的编写思路。

### （二）第二步：基础布局搭建

首先编写侧边栏（sidebar）、头部（header）和主内容区（main-content）的 HTML 容器结构。
在 CSS 文件中运用 Grid 相关属性，完成页面的基础布局框架。

### （三）第三步：嵌套布局开发

分区域逐步开发，在 HTML 中为父容器添加嵌套的子元素。记住，你可以在 Grid 容器内部继续嵌套新的 Grid 容器。

侧边栏部分：使用 Grid 布局规划导航栏和品牌标识区的位置。

头部部分：使用 Grid 布局排列搜索栏、用户信息和功能按钮。

主内容区部分：使用 Grid 布局划分项目展示区、公告区和热门推荐区。

填充测试内容和占位图片，以此调整所有 Grid 项目的位置。

### （四）第四步：素材资源收集

完成 Grid 布局后，你既可以复刻本项目提供的仪表盘样例，也可以自主设计个性化样式。

所有所需图标均可从 [Material Design Icons](https://pictogrammers.com/library/mdi/) 下载 SVG 格式文件。

自主选择字体！设计样例中使用的是 Roboto 字体，该字体可在 Google Fonts 中获取。（你可以回顾《[文字样式进阶](https://www.odincode.com/course/web/html-css-advanced/section/text-styles-advanced)》课程，了解如何在项目中引入外部字体）。

### （五）第五步：实用小贴士

搭建布局的过程中，可以给容器添加背景色或边框，便于直观查看 Grid 布局的效果。

定义 Grid 轨道时，你可以自行决定使用像素（px）、比例单位（fr），或两种单位结合使用。

本项目不强制要求响应式设计，但如果你有兴趣，可以实现浏览器窗口缩放时，项目卡片区域同步伸缩的效果。

无需做到和设计样例像素级一致，把这个项目当作锻炼 CSS 技能、发挥创意的机会就好。

别忘了将完成的仪表盘项目推送到 GitHub，再通过 GitHub Pages 把它发布到线上，让更多人看到你的作品！

### （六）第六步：课程反馈提交

在进入下一阶段学习前，诚邀你填写问卷，分享对《HTML 和 CSS 进阶》课程的反馈。
